# squash-cli

A command-line tool to interactively squash, reorder, and reword git commits before pushing.

## Overview

`squash-cli` provides an interactive terminal UI for managing your local git commit history. Before you push your branch, you can use it to clean up messy work-in-progress commits — squashing them together, reordering them, or rewording their messages — without needing to remember `git rebase -i` syntax.

## Features

- Interactively squash multiple commits into one
- Reorder commits in your branch history
- Reword commit messages
- Terminal UI powered by [Inquirer.js](https://github.com/SBoudrias/Inquirer.js)
- Colorized output via [Chalk](https://github.com/chalk/chalk)

## Requirements

- Node.js >= 18.0.0
- Git installed and available on your `PATH`

## Installation

Install globally via npm:

```bash
npm install -g squash-cli
```

Or use it without installing via npx:

```bash
npx squash-cli
```

## Usage

Run the `squash` command from inside a git repository:

```bash
squash
```

The interactive prompt will guide you through selecting commits to squash, reorder, or reword before you push your branch to a remote.

## Development

Clone the repository and install dependencies:

```bash
git clone https://github.com/psenger/squash-cli.git
cd squash-cli
npm install
```

Run in development mode (uses `ts-node`, no build step required):

```bash
npm run dev
```

Build the TypeScript source to `dist/`:

```bash
npm run build
```

Run tests:

```bash
npm test
```

Run tests with coverage:

```bash
npm run test:coverage
```

Lint the source:

```bash
npm run lint
```
